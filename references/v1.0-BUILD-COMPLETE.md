# v1.0 Build Complete

The first stable Evo QA build, in five phases.

## Phase 1 — Doctor

Environment health checks: Python version, Playwright install, browser binaries, write permissions, workspace defaults. 12 checks, all green on a clean install.

CLI: `python -m evo_qa.core.cli doctor` / `setup`.

## Phase 2 — Knowledge

Three-layer knowledge architecture (universal / industry / project), markdown frontmatter as the storage format, scope/tag/priority indexing, grep-based retriever as the v1.0 default (BM25/vector reserved for future).

Bundled corpus: ~48 entries across:
- **universal heuristics** — SFDIPOT, Goldilocks, MIDTESTD, SLIME, DUFFSSCRA, W5HE…
- **test-design heuristics** — FCC CUTS VIDS, RIMGEA, RCRCRC, FIBLOTS, HICCUPPS, FEW HICCUPPS, I SLICED UP FUN, IVECTRAS, Kaner attacks, CRUSSPIC STMPL, Rapid App-Level…
- **data heuristics** — boundaries, attack cheatsheet, FLOOD, input methods, starvation
- **exploration** — SBTM charter format, McOaster report template
- **selector pitfalls** — virtualized dropdown gotcha
- **glossary** — ISTQB CTFL v4 vocabulary, QA mnemonic index
- **domain glossaries**

## Phase 3 — Atoms + Retry + Watchdog

12 action atoms (`navigate`, `click`, `fill`, `press`, `select_option`, `wait_for`, `expect_visible`, `expect_text`, `screenshot`, `evaluate`, `dispatch_event`, `dropdown_select`).

Three-layer retry:
- `physical` — 3 retries on transient errors per atom
- `heal` — 1 selector-healing attempt per step
- `case_budget` — 10 retries per case
- per-case `timeout=600s` hard ceiling

Watchdog has two modes:
- `soft` — required for Playwright sync (greenlet loop must own the main thread)
- `threaded` — for the async API

## Phase 4 — Curator + Forgetting

Knowledge state machine: `active → stale → deprecated → archived`. Honeymoon grace of 60 days for new entries. `never_archive` tags for `critical` / `compliance` / `regulatory` / `golden`. `revive` command restores an entry and records the revival as a signal.

The forgetting policy is **pseudo-forgetting** — entries flow through states but are never silently deleted. `_archived/` keeps everything diff-able.

## Phase 5 — HTML Report + E2E

`adapters/reporters/html/` renders a self-contained HTML report:
- Configurable color scheme
- System font stack with fallbacks
- Favicon, no logo
- English-only

Generates HTML. (PDF generation via WeasyPrint was removed in v1.0.3 — use the browser's Print → Save as PDF instead.)

## v1.0.1 — Smart Dropdown Patch

Five-tier dropdown fallback for the most painful real-world widget:
1. native `<select>`
2. ARIA Combobox role
3. typeahead pattern
4. virtualized scroll
5. role + text matching

Validated against four carrier-template mocks. New atom: `dropdown_select`.

## v1.0.2 — RunBus Decoupling

See `architecture-lesson-runbus.md` for the full story.

The reverse-feedback layer (brain extraction, curation, reflection) was extracted from the main flow into:

- `core/run_bus.py` — append-only `events.jsonl` + per-sink cursors + dead-letter
- `core/sinks/` — `Sink` ABC + `ExtractorSink` (idempotent, isolated)
- `core/scheduler/` — `Job` ABC + `CuratorJob` (24h hygiene) + `ReflectionJob` (count-triggered LLM stub) + zero-copy `Snapshot`
- `core/system_brain/writer.py` — `BrainWriter` mediator with field-ownership matrix
- `core/_atomic.py` — `atomic_write` + `append_event` primitives

12 fault-injection tests prove the SLA: the main flow stays green even if every reverse-feedback component is broken.

Three new commands: `curate`, `reflect`, `drain`. Total: 18 commands.

## Architecture summary

Six layers + an async sidecar:

```
L6 SKILL (this file's parent)
  └── L5 Orchestrator (core/orchestrator.py — cmd_*)
       └── L4 Adapters (browsers / executors / ingestors / retrievers / reporters)
            └── L3 Knowledge (knowledge/ + brain/)
                 └── L2 Execution (atoms + retry + watchdog)
                      └── L1 Schemas (typed contracts)

Sidecar (independent of main flow):
  RunBus (events.jsonl, lock-free) → Sinks → Brain (auto layer)
                                  → Jobs → Curator + Reflection → Brain (business layer, _proposed)
```
